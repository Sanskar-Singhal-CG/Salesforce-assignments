Part 2: Practical Tasks - Step by Step Guide
(Do this in a Salesforce Developer Org: Setup > Flows)

=====================================================
BEFORE YOU START - Create the custom fields you'll need
=====================================================
1. Go to Setup > Object Manager > Opportunity > Fields & Relationships > New.
   - Field type: Text (or Picklist), Field Name: Status, API Name: Status__c
   - Save and add it to the page layout.
   (Note: Opportunity already has a standard "Stage" field, but the assignment
   wants a separate custom field called Status, so create it as above.)

2. Go to Setup > Object Manager > Contact > Fields & Relationships > New.
   - Field type: Text (or Picklist), Field Name: Engagement Level, API Name: Engagement_Level__c
   - Save and add it to the page layout.

=====================================================
TASK 1: Create a Screen Flow (New Contact form)
=====================================================
1. Setup > search "Flows" > click Flows > click New Flow.
2. Choose "Screen Flow" as the flow type > Create.
3. Click the "+" node and add a Screen element.
   - Label it "Contact Details".
   - Drag in three input fields from the left panel: Text (for Name), Email (for Email), Text (for Phone).
   - Give each one an API name, e.g. NameInput, EmailInput, PhoneInput.
4. Click "+" again after the screen and add a "Create Records" element.
   - Object: Contact
   - Set field values:
     LastName = {!NameInput}
     Email = {!EmailInput}
     Phone = {!PhoneInput}
5. Click Save. Name it "New Contact Screen Flow".
6. Click "Activate".
7. Test it by clicking "Run" and filling in the screen - then check the Contacts tab to confirm a new record was created.

=====================================================
TASK 2: Record-Triggered Flow (Opportunity Status)
=====================================================
1. Flows > New Flow > choose "Record-Triggered Flow" > Create.
2. Set:
   - Object: Opportunity
   - Trigger the Flow When: "A record is created"
   - Set Entry Conditions: leave as "None" (or leave default, since it should run for every new Opportunity)
   - Optimize for: "Actions and Related Records" (fast field updates) - this lets you update the
     triggering record directly without an extra Update Records element.
3. Click Done.
4. In the flow canvas, click "+" and add an "Update Records" element (or use the built-in
   field update if you chose fast field update option).
   - Set Status__c = "Initiated"
5. Save as "Opportunity Status Auto-Update" and Activate it.
6. Test by creating a new Opportunity and checking that its Status field shows "Initiated".

=====================================================
TASK 3: Flow with Loop and Assignment (Engagement Level)
=====================================================
1. Flows > New Flow > choose "Record-Triggered Flow" (or Autolaunched Flow if you want to run it manually/on demand) > Create.
2. If Record-Triggered: Object = Account, Trigger when a record is created or updated.
3. Add a "Get Records" element:
   - Object: Contact
   - Filter: AccountId Equals {!$Record.Id}
   - Store: "Automatically store all fields" and make sure "Get multiple records" is selected.
4. Add a "Loop" element:
   - Collection to loop over: the Contacts collection from the Get Records step.
5. Inside the loop, add an "Assignment" element:
   - Set a variable, e.g. {!loopVar.Engagement_Level__c} = "High" (or whatever value/logic you want).
   - (You can also add a collection variable and use "Add" to build an updated list of Contacts.)
6. After the loop ends, add an "Update Records" element (or "Update Records in a Collection")
   to save the updated Contacts collection back to Salesforce.
7. Save as "Update Contact Engagement Level" and Activate.
8. Test by updating/creating an Account with related Contacts and checking their Engagement Level field.

=====================================================
TASK 4: Decision Element (Email Alert for High Opportunity Amount)
=====================================================
1. You can add this into the same Opportunity flow from Task 2, or make a new Record-Triggered
   Flow on Opportunity (trigger: created or updated).
2. Add a "Decision" element after getting/checking the Opportunity record.
   - Outcome name: "Amount High"
   - Condition: {!$Record.Amount} Greater Than 100000
3. On the "Amount High" (true) path, add an "Action" element:
   - Search for "Send Email" action.
   - To Address: the Sales Manager's email (you can hardcode it or use a formula/lookup to the manager's email field).
   - Subject: "High Value Opportunity Alert"
   - Body: something like "Opportunity {!$Record.Name} has an amount over ₹1,00,000."
4. Save and Activate.
5. Test by creating/editing an Opportunity with Amount > 100000 and confirming the email is sent
   (check Setup > Email Log or your inbox if email deliverability is enabled).

=====================================================
BONUS TASK: Scheduled Flow (Weekly Reminder for Open Cases)
=====================================================
1. Flows > New Flow > choose "Schedule-Triggered Flow" > Create.
2. Set:
   - Start Date: any upcoming Monday
   - Frequency: Weekly, repeat every Monday
3. Add a "Get Records" element:
   - Object: Case
   - Filter: Status Not Equal to "Closed" (or IsClosed Equals False)
4. Add a "Loop" element to go through each open Case.
5. Inside the loop, add a "Send Email" action:
   - To Address: {!loopVar.Contact.Email}
   - Subject: "Reminder: You have an open case"
   - Body: "Hi {!loopVar.Contact.Name}, your case {!loopVar.CaseNumber} is still open. We're working on it."
6. Save as "Weekly Open Case Reminder" and Activate.
7. Since it's scheduled, it will run automatically every Monday - you can also test manually
   using "Debug" in Flow Builder to make sure it works before waiting for the real schedule.

=====================================================
GENERAL TIPS
=====================================================
- Always click "Activate" after saving, otherwise the flow won't actually run.
- Use the "Debug" button in Flow Builder to test flows without waiting for real triggers.
- Check "Paused and Failed Flow Interviews" in Setup if something doesn't run as expected.
- Take screenshots of each flow (canvas + a successful test) since assignments usually want proof of working flows.
